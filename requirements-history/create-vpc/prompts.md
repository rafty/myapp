## 1. requirementsの作成: requirements-draft.md => requirements.md

ファイル`requirements/requirements-draft.md`を分析し、本プロジェクトを検査してください。
AWSのプロフェッショナルな視点で、`requirements-draft.md`のドラフト要件を改善し、新しいファイル `requirements/requirements.md`に書き直してください。


## 1.1. requirements-draftの改善: requirements-draft.md => requirements.md
> 期待する要件定義書が作成されない場合

`requirements/requirements.md`の結果から、`requirements-draft.md`を見直しました。
再度、ファイル`requirements/requirements-draft.md`を分析し、
本プロジェクトを検査してください。
AWSのプロフェッショナルな視点で、`requirements-draft.md`のドラフト要件を再検討し`requirements/requirements.md`を改善してください。

## 2. Plan 作成 : requirements.md -> plan.md

`requirements/requirements.md` を分析し、このプロジェクトの改善のための詳細な計画を作成してください。
計画を requirements/plan.md ファイルに記述してください。

## 3. タスクリスト作成 : plan.md -> tasks.md

`requirements/plan.md` に記載されている計画に従って、詳細な列挙型タスクリストを作成してください。
タスク項目には、タスク完了時に完了マーク [x] を付けるためのプレースホルダー [ ] が必要です。
タスクリストを `requirements/tasks.md` ファイルに書き込んでください。

## 4. タスク実行 : tasks.md

タスクリスト `requirements/tasks.md` を完了してください。
`requirements/requirements.md`、`requirements/plan.md`、`requirements/tasks.md`を参照し、すべてのコンテキストを考慮してタスクリスト内のタスクを実装してください。
タスクを順番に完了することに集中してください。
タスクが完了したら、[x] を使用して完了マークを付けてください。
各ステップが完了したら、タスクリストのマークとタスクの完了マーク [x] を更新することが非常に重要です。

## 4.1 タスク実行の継続

タスクリスト`requirements/tasks.md`に完了マークがついていないタスクがあります。
続けてタスク実行を実施してください。
